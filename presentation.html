Architecture Overview

src/
  app/
    page.tsx ← Server Component (dashboard)
    loading.tsx, error.tsx, not-found.tsx
    api/tasks/ ← REST API routes
    tasks/[id]/ ← Detail page
  components/ ← Client Components
    TaskBoard, TaskCard, TaskFilter
    CreateTaskModal, EditTaskForm
    ThemeProvider, ThemeToggle, Toast
  db/ ← Drizzle schema + connection
  types/ ← Shared TypeScript types
  lib/ ← Utility functions
  middleware.ts ← Request logging

Data Flow

  • Dashboard (Server Component) fetches tasks directly from Neon DB
  • Passes data to Client Components for interactivity
  • Mutations go through API routes (/api/tasks)
  • router.refresh() revalidates server data after mutations

Key Technical Decisions

Server vs. Client Components

Pages are Server Components — fetch data at the server, no client-side loading waterfalls, DB credentials stay secure. Interactive parts (board, forms, filters) are Client Components.

Drizzle ORM + Neon

Drizzle gives type-safe queries with inferred TypeScript types from the schema. Neon uses HTTP-based serverless driver — no connection pooling needed, works perfectly with Vercel edge.

URL Search Params for Filtering

Filters are stored in URL params (?status=todo&priority=high). This makes filtered views shareable and bookmarkable. Server reads params and filters before rendering.

Optimistic UI with useOptimistic

Status changes update the UI instantly using React 19's useOptimistic + useTransition. If the API fails, the UI reverts. Toast notifications confirm success/failure.

Challenges & Solutions

Tailwind v4 Dark Mode

The built-in dark variant uses media queries. Switching to class-based required @variant dark (not @custom-variant) to override the built-in variant rather than creating a duplicate.

Next.js 15 Async Params

In Next.js 15+, params and searchParams are now Promises. Had to await them in Server Components — a breaking change from Next.js 14.

Mobile Responsiveness

Filter bar overflowed on small screens. Fixed with full-width search on mobile, 3-column grid for dropdowns, and bottom-sheet modal pattern for the create form.

Test Isolation

React Testing Library renders accumulated across tests in Vitest. Added explicit cleanup() in afterEach to prevent duplicate element errors.
